This is another alterative to my preferred fit method as applied to the linear files. In this case, the data points are not inversely weighted by 1/f.  This gives less weight to the low-frequency data points than in my preferred method.

This fit method is applied to the linear files.  Everything is done using the log(amplitudes) that were provided.

A Brune model is assumed with high-frequency falloff rate of 2.

The data and synthetic are forced to agree at low frequencies, such that they have the same average value (in log amplitude) over points 3, 4, and 5 in the spectrum.  Note that this corresponds to frequencies (1.004, 1.406, 1.808 Hz).

The best-fitting model is obtained from the minimum rms residual (difference between data and prediction) over the 2 to 40 Hz band using a simple grid search over values of corner frequency.  Each data point receives the same weight.

The accompanying fit file shows:
ID, f_c, fitrms

I have not computed formal error estimates, but larger fitrms values are clearly associated with greater uncertainties.
